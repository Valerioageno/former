# Radio

Radio buttons differ from other inputs because they need a wrapper which keeps the state.

That wrapper is provider by Former itself and is the `<RadioGroup />` component. The only
prop it takes is the `name: string`

<Tabs>
<TabItem value="typescript" label="Typescript">

```tsx
import { Form, Radio, RadioGroup, Value } from 'react-former'

function App(): JSX.Element {
  const submit = (e: FormEvent<HTMLFormElement>, data: Record<string, Value>): void => {
    e.preventDefault()
  }

  return (
    <Form onSubmit={submit}>
      <RadioGroup name="radio-group">
        <Radio name="radio1" initialValue={false} />
        <Radio name="radio2" initialValue={true} />
      </RadioGroup>
      <button type="submit">Submit</button>
    </Form>
  )
}
```

</TabItem>
<TabItem value="javascript" label="Javascript">

```jsx
import { Form, Radio, RadioGroup } from 'react-former'

function App() {
  const submit = (e, data) => {
    e.preventDefault()
  }

  return (
    <Form onSubmit={submit}>
      <RadioGroup name="radio-group">
        <Radio name="radio1" initialValue={false} />
        <Radio name="radio2" initialValue={true} />
      </RadioGroup>
      <button type="submit">Submit</button>
    </Form>
  )
}
```

</TabItem>
</Tabs>

The `<Radio />` component takes all the basic HTMLInputAttributes.

Setting the `type` attribute won't have any effect as well as the `checked` attribute which will be overrided by the `initialValue`.
